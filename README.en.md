# Alirator
A bilingual (Arabic/English) static personal site template — zero-build, config-driven, and fully customizable from a single file.

[**العربية**](README.md)

</div>

## Features

- **Bilingual** — Arabic/English with full RTL support
- **Config-driven** — theming, content, and structure from `config.yaml`
- **Markdown content** — static pages, blog posts, and short posts
- **Zero-build** — static site that works directly from files
- **Automatic SEO** — Open Graph tags and page descriptions
- **Syntax highlighting** — via highlight.js
- **Diagrams** — via Mermaid
- **Auto-deploy** — GitHub Actions workflow included

## Quick Start

```bash
# Clone the template
git clone https://github.com/AliOsm/alirator.git my-site
cd my-site

# Edit configuration
# Edit config.yaml with your name, links, and accent color

# Edit pages
# Edit Markdown files in the pages/ directory

# Build manifests and run locally
bash dev.sh
# Open http://localhost:6570
```

> `_manifest.json` files are auto-generated by `build.rb` and gitignored.

## Configuration

`config.yaml` controls everything:

| Section | Description |
|---|---|
| `siteName` | Site name (Arabic/English) |
| `defaultLang` | Default language (`ar` or `en`) |
| `theme.accent` | Primary accent color (hex) |
| `author` | Name, bio, avatar, location, social links |
| `pages` | Static pages — `[slug, Arabic, English]` (use `~` to hide a page from a language) |
| `collections` | Collections — `[slug, Arabic, English, type]` |
| `seo` | Site description and share image |
| `footer` | Footer text (supports `{year}`) |

## Adding Content

### Static Pages

Create a file in `pages/` as `slug.lang.md`, then add it to `pages` in `config.yaml`:

```yaml
pages:
  - [slug, Arabic Title, English Title]  # both languages
  - [slug, عربي فقط, ~]                  # Arabic only
  - [slug, ~, English Only]              # English only
```

Use `~` (YAML null) in place of a title to hide the page from that language.

### Blog Posts and Short Posts

Create a file in `pages/blogs/` or `pages/posts/` with front matter:

```yaml
---
title: Post Title
date: 2024-06-15
excerpt: A brief description.
tags: [tag1, tag2]
cover: /path/to/image.jpg  # optional
---
```

Then run `ruby build.rb` to update the manifests.

## Deployment

### GitHub Pages

This repo includes a [deployment workflow](.github/workflows/deploy.yml). On push to `main`:

1. Builds collection manifests
2. Appends a hash to all files for cache-busting
3. Deploys the site to GitHub Pages

To enable: **Settings → Pages → Source → GitHub Actions**.

### Other Hosting

The site is just static files — it works on any host (Netlify, Vercel, Cloudflare Pages, etc.).

## Local Development

```bash
bash dev.sh
```

Builds manifests and starts a local server on port 6570.

## Dependencies

- **Ruby 2.7+** — for building manifests and the dev server (built-in libraries only, no gems)
- All other libraries loaded via CDN

## Using Other Languages

The template defaults to Arabic and English, but supports any language pair. To switch:

1. Update the language codes in `build.rb` (`extract_slug_and_lang`) and `app.js`
2. Set `langDir` in `config.yaml` to define the direction for each language
3. Change the Google Fonts in `index.html` to match the new languages
